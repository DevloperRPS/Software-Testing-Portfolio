## Environment

* **Operating System:** Ubuntu 24.04 LTS
* **Browser:** Google Chrome (Latest Version)
* **Platform:** Desktop
* **Testing Type:** Manual Functional Testing
* **Build/Test Date:** During QA Technical Assessment


## Severity: **Major**


## Priority: **P1 – High**

## Steps to Reproduce

1. Open the VoiceToNotes dashboard in Google Chrome.
2. Log in using a valid account.
3. Navigate to the voice recording feature.
4. Allow microphone permission and record a short voice note.
5. Stop the recording and wait for the transcription process to complete.
6. Observe the response time before the transcribed text is displayed.


## Expected Result

The application should process the recording and display the transcription within a reasonable response time, providing a smooth user experience.


## Actual Result

The transcription is eventually generated, but there is a noticeable delay before the text appears. 

The response time is longer than expected, making the voice-to-text workflow feel slow and less responsive.


## Conclusion

Although the transcription functionality works, the delayed response affects the overall user experience, particularly for a feature expected to provide near real-time results. 

Improving transcription processing and response time would enhance application performance and usability.
