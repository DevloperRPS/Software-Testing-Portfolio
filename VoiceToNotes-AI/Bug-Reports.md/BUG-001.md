# BUG-001 – Mobile Features Page Displays "Not Supported" Message

<img width="1080" height="2400" alt="mobile-chrome-browser-screenshot-of-voicetonotes-ai-web-page-shoing-error" src="https://github.com/user-attachments/assets/5960300d-89cc-4516-9d63-3e643289c010" />


## Module/Page: **Landing Page – Features Section (Mobile View)**


## Environment

* **Operating System:** Ubuntu 24.04 LTS (Mobile Testing)
* **Browser:** Google Chrome (Latest Version)
* **Platform:** Mobile
* **Testing Type:** Manual Functional Testing
* **Build/Test Date:** During QA Technical Assessment


## Severity: **Major**
## Priority: **P1 – High**


## Steps to Reproduce

1. Open the VoiceToNotes.ai landing page on a mobile device using Google Chrome.
2. Wait for the homepage to load completely.
3. Tap the **Features** section or navigate to the Features page.
4. Observe the page behavior.


## Expected Result

The Features page should load correctly on mobile devices with all content accessible and responsive.


## Actual Result

Instead of displaying the Features page, a **"Not Supported"** message is shown, preventing users from accessing the feature information on mobile devices.


## Conclusion

The Features section is inaccessible on mobile devices due to a "Not Supported" message being displayed. 

This limits user access to important product information and negatively impacts the mobile browsing experience. 

The issue should be investigated to ensure the Features page functions correctly across supported mobile devices.
