# BUG-002 – Test Login Credentials Not Provided for Dashboard Access

## Module/Page: **Authentication / Dashboard Access**

## Environment

* **Operating System:** Ubuntu 24.04 LTS
* **Browser:** Google Chrome (Latest Version)
* **Platform:** Desktop
* **Testing Type:** Manual Functional Testing
* **Build/Test Date:** During QA Technical Assessment

## Severity: **Major**


## Priority: **P1 – High**


## Steps to Reproduce

1. Review the QA Technical Assessment document.
2. Navigate to the VoiceToNotes dashboard login page.
3. Attempt to locate the test account credentials provided for assessment.
4. Verify the assessment instructions for login information.


## Expected Result

The assessment should provide valid test credentials or clear instructions for accessing the dashboard so that all required test scenarios can be executed.


## Actual Result

No test login credentials or access instructions were provided with the assessment, preventing complete verification of dashboard functionality that requires authentication.


## Conclusion

The absence of test login credentials limits the ability to perform complete functional testing of authenticated features. 

Providing a dedicated test account or clear access instructions would ensure that all assessment scenarios can be executed as intended.
