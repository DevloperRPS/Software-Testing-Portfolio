# BUG-003 – Voice Recording Does Not Generate Transcription After Microphone Permission

## Module/Page: **Voice-to-Text Workflow**

## Environment

* **Operating System:** Ubuntu 24.04 LTS
* **Browser:** Google Chrome (Latest Version)
* **Platform:** Desktop
* **Testing Type:** Manual Functional Testing
* **Build/Test Date:** During QA Technical Assessment


## Severity: **Critical**

## Priority: **P0 – Highest**


## Steps to Reproduce

1. Open the VoiceToNotes dashboard in Google Chrome.
2. Log in to the application.
3. Navigate to the voice recording section.
4. Allow microphone permission when prompted.
5. Start recording and speak into the microphone.
6. Stop the recording and wait for the transcription to be generated.


## Expected Result

The application should capture the recorded audio and generate an accurate text transcription within a reasonable time.

## Actual Result

Although microphone permission was granted successfully, the application did not generate any transcription from the recorded audio, preventing completion of the core voice-to-text workflow.

## Conclusion

This issue affects the application's primary functionality by preventing users from converting recorded speech into text. 

Since the core voice-to-text workflow cannot be completed, the defect should be treated as a **Critical (P0)** issue and prioritized before release.
