# BUG-003 – 504 Gateway Timeout Error on Reset Password Request

During manual testing of the OrangeHRM application, an issue was observed while using the **Forgot Password** feature. After clicking the **Reset Password** button, the application displayed a **504 Gateway Timeout** error instead of immediately processing the request.

## Observation

While attempting to reset the account password, clicking the **Reset Password** button resulted in a **504 Gateway Timeout** error, preventing the request from being processed successfully during the testing session.

During the delay, the application displayed a **504 Gateway Timeout** error before eventually processing the request. 

This behavior may interrupt the password recovery process, reduce user confidence, and create the impression that the application is unavailable or unresponsive.

## Recommendation

- Investigate the response time of the **Reset Password** functionality.
- Optimize processing or page loading to reduce the delay.
- Ensure user feedback, such as a loading indicator, is displayed while the request is being processed


## Conclusion

The **Forgot Password** functionality encountered a **504 Gateway Timeout** error when the **Reset Password** button was used during manual testing. 

Since this issue affects a critical account recovery feature, it has been classified as a **High Severity** defect. 

Further investigation is recommended to identify the root cause and ensure the password reset process functions reliably.
