# BUG-004 : Employee Name Input Removes Characters While Editing

During manual testing of the OrangeHRM application, an issue was observed while editing employee names. 

When attempting to modify an existing name, some characters were unexpectedly removed from the input field, resulting in incorrect or incomplete text.

## Observation

While editing an employee's name, the input field did not retain all typed characters consistently. 

Some letters were automatically removed during the editing process.

## Impact

The issue affects data entry by preventing users from editing employee names accurately. 

This may lead to incorrect employee records and reduce confidence in the application's data management functionality.


## Recommendation

- Verify the input field behavior while editing existing employee names.
- Ensure all user-entered characters are retained correctly during text modification.
- Perform cross-browser testing to confirm consistent input handling.


## Conclusion

The employee name editing functionality exhibited inconsistent input behavior during manual testing, causing characters to be removed unexpectedly. 

Resolving this issue would improve data accuracy and provide a more reliable user experience.
