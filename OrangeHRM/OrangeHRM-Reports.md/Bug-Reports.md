# Bug Reports

This directory contains the bug reports prepared during the manual testing of the OrangeHRM web application. 

Each report documents a specific issue identified during the testing cycle, along with its description, reproduction steps, expected and actual results, severity, supporting evidence, and recommendations.

## Objective

The objective of maintaining these bug reports is to provide structured documentation of the issues observed during testing and to facilitate efficient defect tracking and resolution as part of the Software Testing Life Cycle (STLC).

## Bug Reports Included

The following bug reports are included in this directory:

- [**BUG-001:**](BUG-001.md) Employee Profile Picture Not Displayed in Directory Module
- [**BUG-002:**](BUG-002.md) Upgrade Section Does Not Display Pricing Information
- **BUG-003:** Call-to-Action (CTA) Response Delay
- **BUG-004:** Characters Occasionally Removed While Entering Text in Input Fields

## Summary

A total of **four potential software defects** were identified during the manual testing cycle. 

Each bug has been documented using a standardized reporting format, including reproduction steps, expected and actual results, severity, priority, supporting evidence, and recommendations.

These reports demonstrate the defect identification and reporting phase of the Software Testing Life Cycle (STLC) and serve as part of this software testing portfolio.
