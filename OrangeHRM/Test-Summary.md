# Test Summary

This document summarizes the manual testing activities performed on the OrangeHRM web application. 

It provides an overview of the testing scope, execution results, key observations, and the overall assessment of the application's functionality and user experience.


## Objective

The objective of this [testing cycle](Test-Cases.md) was to validate the core functionalities of the OrangeHRM application, identify potential functional issues, and evaluate the application's stability through structured manual testing.
  
## Testing Scope

The testing activities covered the following functional and non-functional areas of the OrangeHRM web application:

- Login & Authentication
- Dashboard Functionality
- Admin Module
- Employee Information Management (PIM)
- Leave Management
- Navigation & Menu Validation
- Form Validation
- Search & Filter Functionality
- Session Management (Login/Logout)
- User Interface & User Experience Assessment

The testing cycle provided valuable insights into the functional behavior and overall quality of the OrangeHRM web application. The following outcomes summarize the testing activities:

- A total of **25 manual test cases** were successfully executed across multiple application modules.
- **Four potential software defects** were identified during the testing cycle and documented as individual bug reports with supporting evidence.
- The application's core functionalities, including authentication, navigation, and employee management, performed reliably in most test scenarios.
- Manual testing helped identify functional issues as well as user experience improvements that may not be detected through automated testing alone.
- Proper documentation of test cases, execution results, and bug reports improved the traceability and quality of the testing process.
- Tool-based validation using performance, accessibility, and security assessment tools complements manual functional testing and provides a broader quality evaluation.
- The project demonstrated the complete workflow of the Software Testing Life Cycle (STLC), from test planning and execution to defect reporting and test documentation.

## Conclusion

The manual testing cycle successfully evaluated the core functionalities of the OrangeHRM web application, including authentication, navigation, employee management, directory operations, and administrative features. 

The majority of the executed test cases produced the expected results, demonstrating that the application is functionally stable across the tested modules.

During the testing process, **four potential software defects** were identified and documented as individual bug reports with detailed reproduction steps, supporting evidence, severity, and recommendations. These findings highlight the importance of systematic testing in identifying areas for functional and usability improvements.

Overall, this testing project demonstrated the complete Software Testing Life Cycle (STLC), including test planning, test case design, execution, defect reporting, and test documentation. 

The project provided valuable hands-on experience in professional QA practices and contributed to building a structured software testing portfolio.
